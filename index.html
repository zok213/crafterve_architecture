This document provides a comprehensive analysis of 4 architecture options for deploying Crafterse's image generation service, focusing on Stable Diffusion XL/Flux integration with Kafka routing and Redis caching.

üéØ Key Decision Criteria

  • Phase 1 Priority: Cost = $0, Time to Deploy = 3-5 days, User Load = 5-10 concurrent users
  • Core Requirements: Stable Diffusion XL/Flux integration, Kafka request routing, Redis asset caching, CloudFlare R2 storage
  • Technical Constraints: CPU-only inference, 16GB RAM limit, 100 requests/hour rate limit per Space

üèõÔ∏è Architecture Options Overview

Option 1: FREE TIER

ü§ó HuggingFace Spaces + Kafka

Best for: MVP/Phase 1 development (0-3 months)

  • ‚úÖ Free HuggingFace Spaces (16GB RAM, 2vCPU, 30GB storage)
  • ‚úÖ CompVis/stable-diffusion-xl-base-1.0 running on CPU
  • ‚úÖ Apache Kafka for request routing and load balancing
  • ‚úÖ Redis for generated asset caching (30MB limit)
  • ‚úÖ HuggingFace storage + CloudFlare R2 for asset delivery
  • ‚úÖ CloudFlare for global asset distribution
  • ‚ö†Ô∏è Limitations: CPU-only inference, 60s cold start, 100 req/hour limit
Option 2: PRODUCTION

‚ö° Replicate API + Managed Services

Best for: Phase 2 expansion (3-12 months, 10-100 users)

  • ‚úÖ Replicate API for GPU-accelerated model inference
  • ‚úÖ AWS SQS for request queuing and processing
  • ‚úÖ AWS ElastiCache (Redis) for asset caching
  • ‚úÖ AWS S3 for asset storage with lifecycle policies
  • ‚úÖ CloudFlare CDN for global asset delivery
  • ‚úÖ AWS CloudWatch for performance tracking
  • ‚úÖ Cost: $70-100/month (based on 30K images/month)
  • ‚úÖ Advantages: GPU acceleration, 15-30s generation time, 99.9% uptime SLA
Option 3: CUSTOM

‚ò∏Ô∏è Custom Kubernetes Deployment

Best for: Phase 3 production (12+ months, 100+ users)

  • ‚úÖ Self-managed Kubernetes cluster with GPU nodes
  • ‚úÖ Custom Stable Diffusion deployment with optimizations
  • ‚úÖ Apache Kafka cluster for high-throughput processing
  • ‚úÖ Redis Cluster with persistence and replication
  • ‚úÖ Multi-cloud storage with automated backup
  • ‚úÖ Prometheus + Grafana stack with custom dashboards
  • ‚úÖ GitHub Actions for automated deployments
  • ‚úÖ Cost: $200-500/month (optimized for MVP)
  • ‚úÖ Cost Optimizations: Cheaper GPU (g4dn vs p3.2xl), fewer nodes, free monitoring
  • ‚úÖ Advantages: Full control, custom optimizations, auto-scaling
Option 4: HYBRID

üîÑ Hybrid Approach (Spaces + API Fallback)

Best for: Transition periods and cost optimization

  • ‚úÖ Primary: HuggingFace Spaces for low-traffic periods
  • ‚úÖ Fallback: Replicate API for peak loads and GPU acceleration
  • ‚úÖ Kafka with intelligent routing based on load
  • ‚úÖ Multi-level caching (Redis + in-memory)
  • ‚úÖ Hybrid storage strategy with failover
  • ‚úÖ Custom routing logic based on queue depth
  • ‚úÖ Cost: $35-55/month (2 Spaces + 15K API calls)
  • ‚úÖ Advantages: Best of both worlds, cost optimization

Data Flow Architecture

User Request ‚Üí Crafterse Backend ‚Üí Kafka Queue ‚Üí Model Inference ‚Üí Redis Cache ‚Üí CloudFlare R2/CDN ‚Üí Response

1. User submits prompt via game UI
2. Backend validates request and forwards to Kafka
3. Kafka routes to appropriate inference service
4. Model generates image (with GPU acceleration where available)
5. Generated asset cached in Redis
6. Asset uploaded to CloudFlare R2 and CDN
7. URL returned to user via backend
